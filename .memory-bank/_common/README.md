# _common/ — Руководства уровня проекта

Эта папка хранит знания, применимые ко **всему проекту**, а не к одной задаче.

---

## 1. Project Dev Guide — главный документ

**Файл:** `.memory-bank/_common/dev-guide.md`

Единственный инженерный гайд проекта. Накапливается по мере задач — не пересоздаётся.

**Когда создавать / дополнять:**
- При первой инициализации стека (tech-lead создаёт документ).
- При изменении стека, команд, conventions или переменных окружения (tech-lead дополняет).

## 2. Task-Specific Dev Guide — исключение

**Файл:** `.memory-bank/tasks/<TASK-ID>/dev-guide.md`

Создаётся **только** если инструкция применима **исключительно** к данной задаче и никогда не обобщится на весь проект.

Примеры допустимых случаев:
- Одноразовая миграция данных с нестандартными шагами.
- Экспериментальный подход, не принятый как стандарт.
- Воспроизведение специфического бага (шаги актуальны только для него).

**Обязательные ссылки:**
- В начале задачного гайда: `← [Project Dev Guide](.memory-bank/_common/dev-guide.md)`
- В конце общего гайда (секция `## Task-Specific Guides`): ссылка на задачный гайд.

## 3. Technology Guides — паттерны и антипаттерны

**Файл:** `.memory-bank/_common/<technology>/guide.md`

Добавляй когда:
- Обнаружен паттерн, антипаттерн или нюанс, специфичный для технологии.
- Настройка технологии требует нестандартных шагов.
- Один и тот же вопрос возникает повторно.

Операция: UPSERT-KNOWLEDGE через skill memory-bank-owner с параметром:
  path: .memory-bank/_common/<technology>/guide.md

После добавления — обновить index.md через WRITE-ARTIFACT.

**Шаблон:**

```markdown
---
type: tech-guide
technology: <name>
version: 1.0.0
---
# <Technology> Guide

## Problem
<Что за проблема или нюанс>

## Solution
<Как решать>

## Example
```<lang>
<пример кода или команды>
```

## Enforcement
<Что проверяет code-reviewer или architect>
```

## Принципы гайдов
- Гайд универсален: не содержит доменной специфики конкретного проекта.
- Гайд атомарен: одна проблема — один файл.
- Гайд актуален: устаревший гайд хуже отсутствующего — удаляй или обновляй.
